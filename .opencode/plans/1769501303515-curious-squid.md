# Plan: Use PostgreSQL 16+ `ON UPDATE NOW()` for auto-updating timestamps

## Goal
Replace the trigger-based `updated_at` auto-update with PostgreSQL 16+'s `ON UPDATE NOW()` column attribute.

## Changes

### 1. `backend/migration/003_create_feed_entries.sql`
- Remove the trigger function and trigger
- Update `updated_at` column to use `ON UPDATE NOW()`:
  ```sql
  updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW() ON UPDATE NOW()
  ```

### 2. `backend/internal/repository/feed_repository.go:462-478`
- Remove `updated_at` from the UPDATE SET in the upsert query since the column auto-updates:
  ```sql
  ON CONFLICT (policy_document_id) DO UPDATE SET
      source_type     = EXCLUDED.source_type,
      title           = EXCLUDED.title,
      short_text      = EXCLUDED.short_text,
      key_points      = EXCLUDED.key_points,
      political_score = EXCLUDED.political_score,
      impact_score    = EXCLUDED.impact_score,
      source_url      = EXCLUDED.source_url,
      published_at    = EXCLUDED.published_at
      -- no updated_at here!
  ```
